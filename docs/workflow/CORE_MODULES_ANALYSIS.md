# WMS系统核心模块功能分析

## 1. 系统架构概述

WMS系统采用前后端分离架构，主要包括以下核心模块：

### 1.1 后端模块
- **用户认证模块**：处理用户登录、注册、权限管理
- **辅料管理模块**：管理辅料的基本信息、分类、图片等
- **库存管理模块**：管理辅料的库存状态、预警等
- **项目方案模块**：管理项目的方案设计、物料清单等
- **需求单模块**：管理采购需求的创建、审核、执行等
- **采购管理模块**：管理采购订单的创建、跟踪、执行等
- **供应商管理模块**：管理供应商的基本信息、产品等
- **数据统计模块**：提供系统使用数据的统计和分析
- **相似辅料推荐模块**：基于向量数据库的辅料推荐功能
- **系统管理模块**：管理系统的配置、用户、通知等

### 1.2 前端模块
- **登录模块**：用户登录界面
- **仪表盘**：系统概览、热门辅料推荐
- **辅料库**：辅料的浏览、搜索、管理
- **库存管理**：库存状态的查看、预警管理
- **项目方案**：项目方案的创建、管理
- **需求管理**：需求单的创建、审核、管理
- **采购管理**：采购订单的创建、跟踪
- **供应商管理**：供应商信息的管理
- **数据统计**：系统使用数据的统计分析
- **系统管理**：用户、角色、配置管理

## 2. 核心模块功能分析

### 2.1 用户认证模块

**功能描述**：
- 用户登录、注册
- 权限管理
- JWT令牌生成和验证

**输入**：
- 登录：用户名、密码
- 注册：用户名、密码、邮箱等

**输出**：
- 登录成功：JWT令牌、用户信息
- 注册成功：用户创建结果

**内部工作流程**：
1. 用户输入登录信息
2. 前端调用登录API
3. 后端验证用户名和密码
4. 生成JWT令牌
5. 返回令牌和用户信息
6. 前端存储令牌并跳转到主界面

### 2.2 辅料管理模块

**功能描述**：
- 辅料的创建、编辑、删除
- 辅料分类管理
- 辅料图片上传和管理
- 辅料信息的查询和搜索

**输入**：
- 辅料基本信息：名称、代码、价格、分类等
- 辅料图片

**输出**：
- 辅料创建/编辑结果
- 辅料列表
- 辅料详情

**内部工作流程**：
1. 用户在前端创建/编辑辅料
2. 上传图片到MinIO
3. 后端保存辅料信息到MySQL
4. 生成辅料向量并存储到Milvus
5. 返回操作结果

### 2.3 库存管理模块

**功能描述**：
- 库存状态的查看和管理
- 库存预警设置和管理
- 库存变动记录

**输入**：
- 库存预警阈值
- 库存调整信息

**输出**：
- 库存状态列表
- 库存预警信息
- 库存变动记录

**内部工作流程**：
1. 系统定期检查库存状态
2. 当库存低于预警阈值时生成预警
3. 用户查看库存状态和预警
4. 进行库存调整操作

### 2.4 项目方案模块

**功能描述**：
- 项目方案的创建、编辑、删除
- 方案物料清单的管理
- 方案成本计算

**输入**：
- 项目基本信息：名称、客户、截止日期等
- 方案物料清单：辅料、数量、价格等

**输出**：
- 项目方案列表
- 方案详情
- 方案成本

**内部工作流程**：
1. 用户创建项目方案
2. 添加方案物料清单
3. 系统计算方案成本
4. 保存方案信息到数据库

### 2.5 需求单模块

**功能描述**：
- 需求单的创建、编辑、删除
- 需求单的审核流程
- 需求单的执行状态跟踪

**输入**：
- 需求单基本信息：项目、类型、截止日期等
- 需求物料清单：辅料、数量等

**输出**：
- 需求单列表
- 需求单详情
- 需求单状态

**内部工作流程**：
1. 用户创建需求单
2. 提交审核
3. 审核人员审核需求单
4. 审核通过后执行采购流程

### 2.6 采购管理模块

**功能描述**：
- 采购订单的创建、编辑、删除
- 采购订单的执行跟踪
- 采购催单管理

**输入**：
- 采购订单基本信息：供应商、日期、状态等
- 采购物料清单：辅料、数量、价格等

**输出**：
- 采购订单列表
- 采购订单详情
- 采购执行状态

**内部工作流程**：
1. 根据需求单创建采购订单
2. 发送采购订单给供应商
3. 跟踪采购执行状态
4. 处理采购催单

### 2.7 供应商管理模块

**功能描述**：
- 供应商的创建、编辑、删除
- 供应商产品的管理
- 供应商评价和评级

**输入**：
- 供应商基本信息：名称、联系人、联系方式等
- 供应商产品信息：产品名称、价格、库存等

**输出**：
- 供应商列表
- 供应商详情
- 供应商产品列表

**内部工作流程**：
1. 用户创建供应商信息
2. 添加供应商产品
3. 系统记录供应商评价
4. 供应商产品与辅料关联

### 2.8 数据统计模块

**功能描述**：
- 系统使用数据的统计
- 辅料使用情况分析
- 采购数据统计
- 图表展示

**输入**：
- 统计时间范围
- 统计维度

**输出**：
- 统计数据
- 图表展示

**内部工作流程**：
1. 用户选择统计参数
2. 系统查询相关数据
3. 生成统计结果和图表
4. 展示统计数据

### 2.9 相似辅料推荐模块

**功能描述**：
- 基于向量相似度的辅料推荐
- 推荐结果的展示
- 推荐理由的说明

**输入**：
- 辅料ID
- 推荐数量

**输出**：
- 相似辅料列表
- 相似度评分

**内部工作流程**：
1. 用户选择辅料
2. 系统从Milvus查询相似辅料
3. 计算相似度评分
4. 展示推荐结果

### 2.10 系统管理模块

**功能描述**：
- 用户管理
- 角色权限管理
- 系统配置管理
- 通知管理

**输入**：
- 用户信息
- 角色权限设置
- 系统配置参数
- 通知内容

**输出**：
- 操作结果
- 系统配置状态
- 通知列表

**内部工作流程**：
1. 管理员进行系统配置
2. 管理用户和角色权限
3. 发送系统通知
4. 监控系统状态
