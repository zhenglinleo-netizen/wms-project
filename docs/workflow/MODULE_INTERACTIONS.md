# WMS系统模块联动关系分析

## 1. 模块联动概述

WMS系统的各个核心模块之间存在密切的联动关系，这些关系构成了系统的整体工作流程。本文档详细分析模块间的调用关系、数据流向和事件触发机制。

## 2. 模块联动关系分析

### 2.1 用户认证与其他模块的联动

**调用关系**：
- 用户认证模块是所有其他模块的基础，所有模块都需要通过用户认证模块进行身份验证
- 其他模块通过JWT令牌验证用户身份和权限

**数据流向**：
- 用户认证模块 → 其他模块：JWT令牌、用户信息
- 其他模块 → 用户认证模块：登录请求、注册请求

**事件触发**：
- 用户登录成功 → 生成JWT令牌 → 跳转到主界面
- 用户权限变更 → 影响其他模块的访问权限

### 2.2 辅料管理与其他模块的联动

**调用关系**：
- 辅料管理模块 → 库存管理模块：辅料库存更新
- 辅料管理模块 → 相似辅料推荐模块：辅料向量生成
- 辅料管理模块 → 供应商管理模块：辅料与供应商产品关联
- 项目方案模块 → 辅料管理模块：查询辅料信息
- 需求单模块 → 辅料管理模块：查询辅料信息
- 采购管理模块 → 辅料管理模块：查询辅料信息

**数据流向**：
- 辅料管理模块 → 库存管理模块：辅料ID、库存数量
- 辅料管理模块 → 相似辅料推荐模块：辅料信息、图片
- 辅料管理模块 → 供应商管理模块：辅料ID、供应商产品ID
- 项目方案模块 → 辅料管理模块：辅料查询条件
- 需求单模块 → 辅料管理模块：辅料查询条件
- 采购管理模块 → 辅料管理模块：辅料查询条件

**事件触发**：
- 辅料创建/编辑 → 生成辅料向量 → 更新Milvus
- 辅料库存变更 → 触发库存预警检查
- 辅料与供应商产品关联 → 更新供应商产品信息

### 2.3 库存管理与其他模块的联动

**调用关系**：
- 库存管理模块 → 需求单模块：库存状态检查
- 库存管理模块 → 采购管理模块：库存状态检查
- 辅料管理模块 → 库存管理模块：辅料库存更新
- 采购管理模块 → 库存管理模块：采购入库

**数据流向**：
- 库存管理模块 → 需求单模块：库存状态
- 库存管理模块 → 采购管理模块：库存状态
- 辅料管理模块 → 库存管理模块：辅料ID、库存数量
- 采购管理模块 → 库存管理模块：采购数量、辅料ID

**事件触发**：
- 库存低于预警阈值 → 生成库存预警
- 采购入库 → 更新库存状态
- 需求单执行 → 扣除库存

### 2.4 项目方案与其他模块的联动

**调用关系**：
- 项目方案模块 → 辅料管理模块：查询辅料信息
- 项目方案模块 → 需求单模块：创建需求单
- 需求单模块 → 项目方案模块：关联需求单

**数据流向**：
- 项目方案模块 → 辅料管理模块：辅料查询条件
- 项目方案模块 → 需求单模块：项目信息、物料清单
- 需求单模块 → 项目方案模块：需求单ID、项目ID

**事件触发**：
- 项目方案创建 → 计算方案成本
- 项目方案物料清单变更 → 重新计算方案成本
- 项目方案审批通过 → 触发需求单创建

### 2.5 需求单与其他模块的联动

**调用关系**：
- 需求单模块 → 项目方案模块：关联项目方案
- 需求单模块 → 库存管理模块：检查库存
- 需求单模块 → 采购管理模块：创建采购订单
- 采购管理模块 → 需求单模块：更新需求单状态

**数据流向**：
- 需求单模块 → 项目方案模块：项目ID
- 需求单模块 → 库存管理模块：辅料ID、数量
- 需求单模块 → 采购管理模块：需求单ID、物料清单
- 采购管理模块 → 需求单模块：采购订单ID、执行状态

**事件触发**：
- 需求单创建 → 检查库存状态
- 需求单审核通过 → 触发采购订单创建
- 需求单执行完成 → 更新需求单状态

### 2.6 采购管理与其他模块的联动

**调用关系**：
- 采购管理模块 → 需求单模块：更新需求单状态
- 采购管理模块 → 库存管理模块：采购入库
- 采购管理模块 → 供应商管理模块：查询供应商信息
- 需求单模块 → 采购管理模块：创建采购订单

**数据流向**：
- 采购管理模块 → 需求单模块：采购订单ID、执行状态
- 采购管理模块 → 库存管理模块：采购数量、辅料ID
- 采购管理模块 → 供应商管理模块：供应商查询条件
- 需求单模块 → 采购管理模块：需求单ID、物料清单

**事件触发**：
- 采购订单创建 → 发送采购订单给供应商
- 采购订单执行 → 更新采购状态
- 采购入库 → 更新库存状态
- 采购完成 → 更新需求单状态

### 2.7 供应商管理与其他模块的联动

**调用关系**：
- 供应商管理模块 → 采购管理模块：供应商信息
- 采购管理模块 → 供应商管理模块：查询供应商信息
- 辅料管理模块 → 供应商管理模块：辅料与供应商产品关联

**数据流向**：
- 供应商管理模块 → 采购管理模块：供应商ID、联系方式
- 采购管理模块 → 供应商管理模块：供应商查询条件
- 辅料管理模块 → 供应商管理模块：辅料ID、供应商产品ID

**事件触发**：
- 供应商信息变更 → 更新相关采购订单
- 供应商产品添加 → 与辅料关联
- 供应商评价更新 → 影响供应商评级

### 2.8 数据统计与其他模块的联动

**调用关系**：
- 数据统计模块 → 辅料管理模块：查询辅料使用情况
- 数据统计模块 → 库存管理模块：查询库存状态
- 数据统计模块 → 采购管理模块：查询采购数据
- 数据统计模块 → 需求单模块：查询需求数据
- 数据统计模块 → 项目方案模块：查询项目数据

**数据流向**：
- 数据统计模块 → 辅料管理模块：统计时间范围
- 数据统计模块 → 库存管理模块：统计时间范围
- 数据统计模块 → 采购管理模块：统计时间范围
- 数据统计模块 → 需求单模块：统计时间范围
- 数据统计模块 → 项目方案模块：统计时间范围

**事件触发**：
- 用户选择统计参数 → 触发数据统计
- 统计数据生成 → 展示统计结果和图表

### 2.9 相似辅料推荐与其他模块的联动

**调用关系**：
- 相似辅料推荐模块 → 辅料管理模块：查询辅料详情
- 相似辅料推荐模块 → Milvus服务：查询相似向量
- 辅料管理模块 → 相似辅料推荐模块：辅料向量生成

**数据流向**：
- 相似辅料推荐模块 → 辅料管理模块：辅料ID
- 相似辅料推荐模块 → Milvus服务：辅料向量
- 辅料管理模块 → 相似辅料推荐模块：辅料信息、图片

**事件触发**：
- 辅料创建/编辑 → 生成辅料向量 → 存储到Milvus
- 用户选择辅料 → 触发相似辅料推荐
- 相似辅料查询 → 展示推荐结果

### 2.10 系统管理与其他模块的联动

**调用关系**：
- 系统管理模块 → 所有其他模块：配置管理、权限管理
- 所有其他模块 → 系统管理模块：日志记录、通知发送

**数据流向**：
- 系统管理模块 → 所有其他模块：系统配置、用户权限
- 所有其他模块 → 系统管理模块：操作日志、通知请求

**事件触发**：
- 系统配置变更 → 影响所有模块的行为
- 用户权限变更 → 影响用户对模块的访问权限
- 系统通知发送 → 所有模块接收通知

## 3. 核心业务流程联动

### 3.1 辅料管理流程

1. **辅料创建**：
   - 用户在前端创建辅料
   - 前端上传图片到MinIO
   - 后端保存辅料信息到MySQL
   - 生成辅料向量并存储到Milvus
   - 更新库存信息

2. **辅料查询**：
   - 用户在前端搜索辅料
   - 前端调用辅料查询API
   - 后端从MySQL查询辅料信息
   - 前端展示辅料列表

3. **相似辅料推荐**：
   - 用户选择辅料
   - 前端调用相似辅料推荐API
   - 后端从Milvus查询相似辅料
   - 计算相似度评分
   - 前端展示推荐结果

### 3.2 项目方案流程

1. **方案创建**：
   - 用户在前端创建项目方案
   - 添加方案物料清单
   - 系统计算方案成本
   - 保存方案信息到数据库

2. **方案审批**：
   - 方案提交审批
   - 审批人员审核方案
   - 审批通过后触发需求单创建

### 3.3 需求单流程

1. **需求单创建**：
   - 用户在前端创建需求单
   - 添加需求物料清单
   - 提交审核

2. **需求单审核**：
   - 审核人员审核需求单
   - 审核通过后触发采购订单创建

3. **需求单执行**：
   - 采购订单执行完成
   - 更新需求单状态
   - 扣除库存

### 3.4 采购管理流程

1. **采购订单创建**：
   - 根据需求单创建采购订单
   - 选择供应商
   - 发送采购订单给供应商

2. **采购执行**：
   - 跟踪采购执行状态
   - 处理采购催单
   - 采购入库

3. **采购完成**：
   - 更新采购订单状态
   - 更新需求单状态
   - 更新库存状态

### 3.5 库存管理流程

1. **库存检查**：
   - 系统定期检查库存状态
   - 当库存低于预警阈值时生成预警

2. **库存调整**：
   - 用户进行库存调整操作
   - 更新库存信息
   - 记录库存变动

3. **库存预警**：
   - 生成库存预警
   - 通知相关人员
   - 触发采购流程

## 5. 总结

WMS系统的各个核心模块之间存在密切的联动关系，这些关系构成了系统的整体工作流程。通过本分析，我们可以看到：

1. **用户认证模块**是所有其他模块的基础，所有模块都需要通过用户认证模块进行身份验证。

2. **辅料管理模块**是系统的核心模块之一，与多个模块存在联动关系，包括库存管理、相似辅料推荐、供应商管理等。

3. **项目方案、需求单、采购管理**三个模块形成了一个完整的业务流程，从项目方案的创建到需求单的生成，再到采购订单的执行。

4. **库存管理模块**与多个模块存在联动关系，包括辅料管理、需求单、采购管理等，负责监控和管理系统的库存状态。

5. **供应商管理模块**与采购管理模块密切相关，负责管理供应商的信息和产品。

6. **数据统计模块**与所有业务模块存在联动关系，负责统计和分析系统的使用数据。

7. **相似辅料推荐模块**与辅料管理模块密切相关，基于向量数据库提供辅料推荐功能。

8. **系统管理模块**与所有其他模块存在联动关系，负责系统的配置、用户管理、权限管理等。

这些模块间的联动关系共同构成了WMS系统的完整工作流程，确保系统能够高效、稳定地运行。
