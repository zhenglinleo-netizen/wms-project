# 库存预警卡片 - 产品需求文档

## 概述
- **Summary**: 将概览页面中的"今日待办/待审任务"卡片修改为库存预警卡片，显示真实的库存预警数据，帮助用户及时了解库存状态。
- **Purpose**: 解决用户无法及时了解库存状态的问题，通过库存预警功能提醒用户关注低库存产品，避免缺货风险。
- **Target Users**: 系统所有用户，特别是采购人员和管理员。

## 目标
- 将"今日待办/待审任务"卡片修改为"库存预警"卡片
- 显示真实的库存预警数据，包括低库存产品数量
- 点击卡片可跳转到库存管理页面
- 当库存预警数据为0时，显示"暂无库存预警"的空状态
- 在管理员的概览界面中将进行中项目卡片替换为显示具体低库存产品列表

## 非目标
- 不修改其他统计卡片的功能
- 不修改其他页面的功能
- 不添加新的API接口，使用现有的产品列表API

## 背景与上下文
- 目前概览页面中的"今日待办/待审任务"卡片显示的是模拟数据，没有实际业务价值
- 库存管理是系统的重要功能，用户需要及时了解库存状态
- 系统已经有产品列表API，可以获取产品的库存信息

## 功能需求
- **FR-1**: 将"今日待办/待审任务"卡片的标题修改为"库存预警"
- **FR-2**: 显示真实的低库存产品数量
- **FR-3**: 点击卡片跳转到库存管理页面
- **FR-4**: 当没有低库存产品时，显示"暂无库存预警"的空状态

## 非功能需求
- **NFR-1**: 卡片样式保持与其他统计卡片一致
- **NFR-2**: 库存预警数据加载速度快，不影响页面整体加载性能
- **NFR-3**: 库存预警逻辑清晰，易于维护

## 约束
- **技术**: 使用现有的产品列表API，不添加新的API接口
- **业务**: 库存预警阈值需要可配置，不同的辅料需要单独设置
- **依赖**: 依赖产品列表API返回的库存数据

## 假设
- 产品列表API返回的数据中包含库存字段和预警阈值字段
- 库存预警阈值可配置是合理的业务需求

## 验收标准

### AC-1: 卡片标题修改
- **Given**: 用户打开概览页面
- **When**: 页面加载完成
- **Then**: 卡片标题显示为"库存预警"
- **Verification**: `human-judgment`

### AC-2: 显示真实库存预警数据
- **Given**: 系统中有低库存产品
- **When**: 用户打开概览页面
- **Then**: 卡片显示低库存产品的数量
- **Verification**: `programmatic`

### AC-3: 点击卡片跳转
- **Given**: 用户点击库存预警卡片
- **When**: 卡片被点击
- **Then**: 跳转到库存管理页面
- **Verification**: `human-judgment`

### AC-4: 空状态显示
- **Given**: 系统中没有低库存产品
- **When**: 用户打开概览页面
- **Then**: 卡片显示"暂无库存预警"
- **Verification**: `human-judgment`

### AC-5: 管理员界面的低库存产品列表
- **Given**: 用户是管理员
- **When**: 打开概览页面
- **Then**: 页面显示具体的低库存产品列表，替换原有的进行中项目卡片
- **Verification**: `human-judgment`
